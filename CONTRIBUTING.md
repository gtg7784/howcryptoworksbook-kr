# 기여 가이드

이 한국어 번역본은 GitHub에서 협력적이고 개방적으로 개발되고 있습니다. 모든 분들의 의견, 기여, 수정을 환영합니다.

> **원본 레포지토리:** [lawmaster10/howcryptoworksbook](https://github.com/lawmaster10/howcryptoworksbook)

## 왜 기여해야 하나요

이것은 암호화폐 시장, 인프라, 리스크를 이해하기 위한 결정적인 고급 자료를 만들기 위한 오픈 소스 노력입니다. 기여함으로써 업계가 차세대 참여자와 빌더를 교육하는 수준을 높이는 데 도움을 주게 됩니다.

## 라이선스와 저작자 표시

원본 저작물은 **CC-BY-NC-ND 라이선스**로 라이선스가 부여되어 있습니다. 이 번역본에 대한 기여는 번역 품질 개선에 초점을 맞추어야 합니다.

## 풀 리퀘스트로 기여하기

이 책에 기여하는 가장 좋은 방법은 풀 리퀘스트를 만드는 것입니다:

1. GitHub 계정으로 **로그인**하거나 새로 만드세요
2. 이 레포지토리를 **포크**하세요. 포크에서 작업하세요.
3. 변경할 **새 브랜치를 생성**하세요, 예: `git checkout -b my_contribution`
4. **파일당 하나의 PR** - 대규모 병합을 피하기 위해 마크다운 파일당 하나의 풀 리퀀스트를 해주세요
5. 변경하고자 하는 **마크다운 파일을 편집**하세요
6. **변경을 커밋**하세요 - 수정 내용을 설명하는 커밋 메시지를 포함하세요
7. 이 레포지토리에 대해 **풀 리퀘스트를 제출**하세요

### 기여할 수 있는 내용

특히 다음을 환영합니다:

- **번역 오류 수정** - 오역, 어색한 표현, 의미 전달 오류
- **기술 용어 개선** - 더 적절한 한국어 기술 용어 제안
- **가독성 향상** - 복잡한 주제의 더 나은 한국어 설명
- **누락된 맥락** - 간과되었을 수 있는 중요한 주의사항이나 고려사항

### 제출하지 말아야 할 것

다음은 PR로 제출하지 마세요:

- 원본 내용에 대한 변경 (원본 레포에 기여해 주세요)
- 명확성을 향상시키지 않는 스타일 선호
- 홍보 콘텐츠나 링크 추가

## 이슈로 기여하기

실수를 발견했지만 어떻게 수정해야 할지 모르거나, 풀 리퀘스트 방법을 모르시면 **이슈**를 등록하세요.

이슈 등록 시 다음을 포함해 주세요:

- 이슈가 나타나는 챕터와 섹션
- 문제에 대한 명확한 설명
- 해당되는 경우, 올바른 번역이 무엇이라고 생각하는지

## 기술 가이드라인

### 줄 끝 처리

모든 제출물은 유닉스 방식의 줄 끝을 사용해야 합니다: **LF** (CR이나 CR/LF가 아님).

### 마크다운 형식

- 표준 GitHub 맛 마크다운 사용
- 일관된 제목 계층 유지 (# 챕터 제목, ## 섹션, ### 하위 섹션)
- 처음 소개되는 핵심 용어에는 **굵게** 사용
- 기술 용어, 주소, 함수 이름 등에는 `코드 서식` 사용

### 커밋 메시지

명확하고 설명적인 커밋 메시지를 작성하세요:

```
좋음: "ch05_custody의 ECDSA 논스 생성 설명 오역 수정"
나쁨: "수정함"
```

## 행동 강령

- 모든 상호작용에서 존중하고 건설적이어야 합니다
- 기여자가 아닌 내용에 집중하세요
- 기여와 피드백에서 선의를 가정하세요
- 토론을 주제에 맞게 전문적으로 유지하세요
